# VK Poll Tracker

CLI-утилита для анализа посещаемости по опросам в беседе VK.
Формирует Excel-отчёт с матрицей «участник × дата» и сводной статистикой.

---

## Требования

- Python 3.11+
- pip

---

## Установка

```bash
pip install -r requirements.txt
```

---

## Настройка `.env`

Скопируйте `.env.example` в `.env` и заполните значения:

```bash
cp .env.example .env
```

| Переменная | Описание |
|---|---|
| `VK_TOKEN` | Токен доступа VK (см. ниже) |
| `VK_API_VERSION` | Версия API, по умолчанию `5.131` |
| `PEER_ID` | ID беседы (см. ниже) |
| `RATE_LIMIT_PER_SEC` | Запросов в секунду, по умолчанию `3` |
| `MAX_CONCURRENT_REQUESTS` | Параллельных запросов, по умолчанию `5` |

### Как получить токен VK

1. Откройте [vkhost.github.io](https://vkhost.github.io/) или используйте [VK OAuth](https://vk.com/dev/implicit_flow_user).
2. Запросите права: `messages` (или `groups` для групповых бесед).
3. Скопируйте `access_token` из адресной строки.

> Альтернатива: создайте standalone-приложение на [vk.com/apps](https://vk.com/apps?act=manage) и получите токен через Implicit Flow.

### Как узнать `PEER_ID` беседы

Для групповой беседы VK:

```
PEER_ID = 2000000000 + локальный_id_беседы
```

Локальный ID беседы можно найти в URL при открытии беседы через браузер:
`https://vk.com/im?sel=c<число>` — это число и есть локальный ID.

Либо вызовите `messages.getConversations` через [VK API Explorer](https://vk.com/dev/methods) и найдите нужную беседу в ответе.

## Запуск

```bash
python -m app.main \
    --date-from 2026-03-01 \
    --date-to 2026-03-31 \
    --output report.xlsx
```

### Параметры

| Параметр | Описание |
|---|---|
| `--date-from` | Начало периода включительно, формат `YYYY-MM-DD` |
| `--date-to` | Конец периода включительно, формат `YYYY-MM-DD` |
| `--output` | Путь к выходному файлу (по умолчанию `report.xlsx`) |

## Логика классификации ответов

| Ключевые слова в тексте варианта | Статус |
|---|---|
| `буду`, `приду`, `да` | `YES` |
| `не`, `нет` | `NO` |
| Всё остальное | игнорируется (участник = `UNKNOWN`) |

Если участник беседы не проголосовал совсем — статус `UNKNOWN`.
